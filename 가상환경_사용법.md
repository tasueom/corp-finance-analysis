# 가상환경(Virtual Environment) 사용 가이드

## 📚 가상환경이란?

**가상환경**은 각 프로젝트마다 독립된 Python 환경을 만드는 도구입니다.

### 왜 필요한가요?

1. **프로젝트별 패키지 관리**: 각 프로젝트마다 필요한 패키지만 설치
2. **버전 충돌 방지**: 프로젝트 A는 numpy 1.0, 프로젝트 B는 numpy 2.0 사용 가능
3. **깔끔한 환경**: 시스템 Python을 건드리지 않아 깨끗하게 유지
4. **재현 가능성**: 다른 컴퓨터에서도 동일한 환경 재현 가능

### 비유로 이해하기

- **시스템 Python** = 아파트의 공용 공간 (모든 사람이 사용)
- **가상환경** = 각 집의 독립된 방 (프로젝트별로 독립)

---

## 🚀 가상환경 사용법

### 1단계: 가상환경 생성

```bash
# Python 3.13을 사용하여 가상환경 생성
py -3.13 -m venv venv
```

또는

```bash
# 기본 Python 버전으로 가상환경 생성
python -m venv venv
```

**설명**: 
- `venv`는 가상환경의 이름입니다 (원하는 이름으로 변경 가능)
- 프로젝트 폴더에 `venv` 폴더가 생성됩니다

---

### 2단계: 가상환경 활성화

#### Windows (PowerShell)
```powershell
.\venv\Scripts\Activate.ps1
```

#### Windows (CMD)
```cmd
venv\Scripts\activate.bat
```

#### Mac/Linux
```bash
source venv/bin/activate
```

**활성화 확인**:
- 터미널 앞에 `(venv)`가 표시되면 성공!
- 예: `(venv) PS C:\...\Team2-corp-finance-analysis>`

---

### 3단계: 패키지 설치

가상환경이 활성화된 상태에서:

```bash
# requirements.txt에 있는 모든 패키지 설치
pip install -r requirements.txt

# 또는 개별 패키지 설치
pip install flask
pip install pandas
```

---

### 4단계: 앱 실행

가상환경이 활성화된 상태에서:

```bash
python app.py
```

---

### 5단계: 가상환경 비활성화

작업이 끝나면:

```bash
deactivate
```

**주의**: 비활성화하면 `(venv)` 표시가 사라지고 시스템 Python으로 돌아갑니다.

## 📋 일상적인 사용 흐름

### 매일 작업할 때:

```bash
# 1. 프로젝트 폴더로 이동
cd "C:\Users\5Class_009\Desktop\Taesu\02 python\Team2-corp-finance-analysis"

# 2. 가상환경 활성화

```

## 🔍 유용한 명령어

### 현재 설치된 패키지 확인
```bash
pip list
```

### 패키지 버전 확인
```bash
pip show flask
```

### requirements.txt 생성 (현재 설치된 패키지 목록 저장)
```bash
pip freeze > requirements.txt
```

### 특정 패키지 제거
```bash
pip uninstall 패키지명
```

## ⚠️ 주의사항

1. **가상환경 폴더는 Git에 올리지 마세요**
   - `.gitignore`에 `venv/` 추가 권장
   - 가상환경은 각자 컴퓨터에서 생성하면 됩니다

2. **가상환경 활성화를 잊지 마세요**
   - 활성화하지 않으면 시스템 Python을 사용하게 됩니다
   - 터미널 앞에 `(venv)` 표시 확인!

3. **새 터미널을 열 때마다 활성화 필요**
   - 터미널을 닫으면 가상환경도 비활성화됩니다
   - 새 터미널에서는 다시 활성화해야 합니다

## 🆘 문제 해결

### PowerShell에서 활성화가 안 될 때

```powershell
# 실행 정책 확인
Get-ExecutionPolicy

# 실행 정책 변경 (관리자 권한 필요)
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

또는 CMD를 사용:
```cmd
venv\Scripts\activate.bat
```

### 가상환경 삭제

```bash
# 비활성화 먼저
deactivate

# venv 폴더 삭제
rmdir /s venv  # Windows
rm -rf venv    # Mac/Linux

# 다시 생성
py -3.13 -m venv venv
```
